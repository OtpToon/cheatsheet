CSS

CSS is used to style a website by giving different html-snippets different attributes.

Typical syntax for Css is:

[Identifier] {
[Attribute]: [Value];
...
}

Identifiers

Identifier Name Description
[name] tag Typical html-tags like h1, p, etc
.[name] class Marked in html with <tag class="[name]">
#[name] id Marked in html with <tag id="[name]">

The tag and class identifiers can be combined to more accuratly decide which elements should be changed

for a class within a tag:

[name] .[name2]{
...
}

for a tag inside a class:

.[name] [name2]{
...
}

Atributes

Attribute Name Valid Values Description
display Display block, inline, inline-block or none (invisible) The display value decides if the element is visible and how it interacts with neighboring elements
position Position static, absolute, fixed, relative, sticky The position value decides the location of the element within the users window.
z-index Z-index auto, number Z-index decides which element will be shown if multiple elements claim the same space.

margin Margin length, %, auto The margin defines the distance neighboring elements need to have from the border of this element.
Margin can be set using 1 (all sides), 2 (top/bottom, sides),3 (top, sides, bottom) or 4 (top, right, bottom, left) Values.
The distances can also be set on their own using margin-top/right/left/bottom.
padding Padding length, % The padding defines the distance from the content of the element to the border of the element.
Padding can be set using 1 (all sides), 2 (top/bottom, sides),3 (top, sides, bottom) or 4 (top, right, bottom, left) Values.
The distances can also be set on their own using padding-top/right/left/bottom.

Values

Valuename Description
block Displays an element as a block element. It starts on a new line, and takes up the whole width
inline Displays an element as an inline element. Any height and width properties will have no effect
inline-block Displays an element as an inline-level block container. The element itself is formatted as an inline element, but you can apply height and width values

static Elements render in order, as they appear in the document flow
absolute The element is positioned relative to its first positioned ancestor element
fixed The element is positioned relative to the browser window
relative The element is positioned relative to its normal position, so "left:20px" adds 20 pixels to the element's LEFT position
sticky The element is positioned based on the user's scroll position

auto The attribute uses a value calculated based on the ancestor-element
% The attribute uses a percentage-value multiplied by the inherited value

length Most popular units: Xpx - X pixels, Xem - X times the font-size of the element
More Units and their description

color Defined using one of the methods below
hex Using a 6-digit hexnumber: #rrggbb
hex with transparency Using a 8-digit hexnumber: #rrggbbaa, opacity increases as the "aa" part increases from #00 to #FF
rgb Using the rgb(red, green, blue) notation with each number ranging from 0 to 255 or a percentage
rgba Using the rgba(reg, green, blue, alpha/opacity) notation where alpha is a number between 0.0 and 1.0
hsl Using the hsl(hue in degree(0-360), saturation in %, lightness in %) notation
hsla Using the hsla(hue in degree(0-360), saturation in %, lightness in %, alpha/opacity as 0.0-1.0) notation

visible The overflow is not clipped. It renders outside the element's box.
clip The overflow is clipped, and the rest of the content will be invisible. Forbids scrolling
scroll The overflow is clipped, but a scroll-bar is added to see the rest of the content

font-sizes Self explaining: medium, xx-small, x-small, small, large, x-large, xx-large
smaller/larger: 1 size larger or smaller than the parent-element
% and length
font-family-type Generic family names are used in case the users browser can't display any other Font. Those are: serif, sans-serif, cursive, fantasy, monospace
